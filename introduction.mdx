## Overview

The Microservicios Ecosystem is a fully containerized, event-driven microservices platform built on **Spring Boot 3.4** and **Java 21**. It demonstrates real-world architectural patterns including hexagonal architecture, JWT-based security, asynchronous messaging, distributed tracing, and centralized log aggregation.

## Services at a Glance

| Service | Port | Description |
|---------|------|-------------|
| **config-server** | 8888 | Centralized configuration (filesystem/native profile) |
| **eureka-server** | 8761 | Service discovery dashboard |
| **auth-service** | 8083 | JWT authentication with BCrypt (HS384) |
| **api-gateway** | 8080 | Spring Cloud Gateway — OAuth2 Resource Server |
| **inventory-service** | 8082 | Product CRUD with hexagonal architecture |
| **notification-service** | 8084 | Async event consumer (RabbitMQ or Kafka) |

## Infrastructure Components

| Component | Port | Role |
|-----------|------|------|
| **RabbitMQ** | 5672 / 15672 | Message broker option 1 |
| **Apache Kafka** | 9092 | Message broker option 2 (KRaft, no Zookeeper) |
| **Zipkin** | 9411 | Distributed tracing UI |
| **Elasticsearch** | 9200 | Log storage and search |
| **Logstash** | 5044 | Log collection and parsing |
| **Kibana** | 5601 | Log visualization UI |

## Prerequisites

- **Java 21**
- **Maven 3.9+**
- **Docker Desktop** (for Docker-based deployment)

<Note>
  Docker Desktop must be running before executing any `docker compose` commands. Check that the system tray icon does not show "starting...".
</Note>
